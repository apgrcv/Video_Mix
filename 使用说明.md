# 视频混剪合并工具使用说明

这是一个用于批量处理视频素材的桌面工具，当前支持三种工作方式：

- `前中后拼接`：适合结构明确的片段组合
- `随机排列组合`：适合从一个素材池中穷举 2 / 3 / 4 段顺序结果
- `加水印`：适合对目录中的所有视频批量叠加图片或文字水印

并且支持可选的拼接转场效果。

---

## 1. 功能概览

### 前中后拼接
- 选择前半段目录
- 可选选择中段目录
- 选择后半段目录
- 程序自动生成所有组合结果

输出数量：
- 无中段时：`前半段数量 × 后半段数量`
- 有中段时：`前半段数量 × 中段数量 × 后半段数量`

### 随机排列组合
- 选择一个统一素材目录
- 选择拼接数量 `2 / 3 / 4`
- 选择拼接顺序：
  - `顺序不同算不同结果`
  - `顺序不同算同一结果`

输出数量：
- 若顺序不同算不同结果：结果总数为 `P(n, k)`
- 若顺序不同算同一结果：结果总数为 `C(n, k)`
- `n` 是素材目录里的视频数
- `k` 是拼接数量

例子：
- 目录里有 4 个视频，拼接数量选 2：
  - 区分顺序：`P(4,2)=12`
  - 不区分顺序：`C(4,2)=6`

说明：
- 区分顺序时，`A+B` 和 `B+A` 是两个不同输出。
- 不区分顺序时，`A+B` 和 `B+A` 视为同一个输出。
- 当前版本虽然叫“随机排列组合”，但实际是穷举生成（不是随机抽样）。

### 转场效果
- 可以在拼接点增加转场
- 默认关闭
- 开启后使用轻量转场
- 轻量转场支持单独选择时长：
  - 0.4 秒
  - 0.6 秒
  - 0.8 秒
  - 1.0 秒

### 加水印
- 选择一个视频目录
- 程序会对目录内所有视频批量加水印
- 支持两种水印来源：
  - PNG 图片水印
  - 多行文字水印
- 支持预览水印位置与大小
- 支持两类显示时间：
  - 指定秒段
  - 结尾倒数秒

## 2. 快速上手（Mac）

### 第一步：初始化环境（只需一次）
在终端进入项目目录后执行：

```bash
sh setup_mac.sh
```

脚本会准备 `bin/ffmpeg` 和 `bin/ffprobe`。

### 第二步：启动程序

```bash
./start.command
```

也可以直接双击 `start.command` 启动。

---

## 3. 快速上手（Windows）

### 第一步：准备 FFmpeg
把 `ffmpeg.exe` 和 `ffprobe.exe` 放到项目根目录的 `bin/` 文件夹中。

### 第二步：启动程序
可选方式：

1. 命令行运行
```bash
python main.py
```

2. 双击运行
- 双击 `start.bat`

## 4. 界面说明

### 顶部 Tab
- `前中后拼接`：默认模式
- `随机排列组合`：排列模式
- `加水印`：批量水印模式
- 鼠标滚轮滚动页面时不会触发这两个 Tab 自动切换

### 素材与输出

#### 前中后拼接 Tab
- `前半段目录`：第一段素材
- `中段目录（可选）`：中间素材
- `后半段目录`：最后一段素材
- `输出目录`：结果保存目录

#### 随机排列组合 Tab
- `文件目录`：统一素材池目录
- `拼接数量`：每个结果由几个视频组成，支持 `2 / 3 / 4`
- `拼接顺序`：
  - `顺序不同算不同结果`：按排列生成（`P(n,k)`）
  - `顺序不同算同一结果`：按组合生成（`C(n,k)`）
- `输出目录`：结果保存目录

#### 目录选择记忆
- 所有“选择”按钮都会记住上一次选择的目录
- 再次点击时，系统目录弹框会优先打开到该目录
- 如果某个字段还没有历史记录，会优先打开最近一次浏览过的目录
- 这些记录保存在项目根目录的 `.video_mix_settings.json`

#### 加水印 Tab
- `视频目录`：要加水印的视频来源目录
- `输出目录`：处理后视频的输出目录
- `水印类型`：
  - `图片水印`
  - `文字水印`
- `图片水印`：
  - 选择 PNG
  - 设置大小百分比、透明度、X/Y
- `文字水印`：
  - 输入多行文本
  - 设置字号、透明度、X/Y
- `预览尺寸`：
  - 输入模拟视频宽高
  - 点击“预览”查看水印在画布中的位置和大小
- `出现时间配置`：
  - 指定秒段（可多条）
  - 结尾倒数秒

### 合并选项
- `优先无损（失败自动转兼容）`
  - 未启用转场时先尝试无损
  - 失败后自动改为兼容拼接
- `强制兼容（统一分辨率）`
  - 所有任务直接重编码

### 转场效果
- `启用转场效果`
  - 默认关闭
  - 开启后会降低速度，但比旧的完整滤镜转场快很多
- `轻量时长`
  - 仅在启用转场时显示
  - 当前可选 `0.4 / 0.6 / 0.8 / 1.0 秒`

### 统一分辨率
- `自动`：以前当前任务中的第一个视频为准
- `自定义`：手动输入宽和高，例如 `1080 x 1920`

### 其他选项
- `跳过已存在文件`：输出目录中已经存在的结果不重复生成
- `并发数`：同时运行的任务数

### 任务进度与日志
- 页面会显示总体完成进度
- 每个线程有独立日志面板
- 日志标题会显示当前文件和 ETA
- 启用转场后，日志会显示当前输出使用的轻量转场时长

## 5. 推荐用法

### 适合用前中后拼接的场景
- 开头、中间、结尾结构明确
- 想控制哪些素材放在前、中、后
- 想批量生成稳定结构的视频组合

### 适合用随机排列组合的场景
- 素材都在一个目录中
- 想测试不同片段顺序的效果
- 想快速生成 2 / 3 / 4 段的顺序排列结果

### 适合开启转场的场景
- 直接硬切看起来太生硬
- 视频片段风格接近，希望衔接更自然
- 你能接受速度变慢，换取更好的观看感受

---

## 6. 性能建议

- `优先无损` 最快，但要求素材参数一致
- 启用转场后仍会比无损拼接慢，但新方案已优先针对速度做优化
- 素材格式较乱时，优先使用 `强制兼容`
- 输出目录尽量放在 SSD
- 并发数建议从 `2-4` 起步，再按机器性能提高
- 随机模式下，如果目录视频很多，输出数量会很大，建议先小规模测试

## 8. 常见问题

### 1. 提示找不到 FFmpeg
- Mac：先执行 `sh setup_mac.sh`
- Windows：确认 `bin/` 下存在 `ffmpeg.exe` 和 `ffprobe.exe`

### 2. 合并失败怎么办
- 查看界面下方日志面板
- 常见原因：文件损坏、参数不兼容、磁盘权限、输出目录不可写

### 3. 无损模式黑屏或异常
- 切换到 `强制兼容（统一分辨率）`
- 兼容模式会重编码并统一输出参数

### 4. 启用转场后为什么还是会变慢
- 因为转场仍然需要先对片段边界做预处理
- 但当前已改成速度优先方案，不再使用之前那种整段重叠滤镜转场

### 5. 随机模式结果数量太多
- 这是正常现象，随机模式是穷举生成
- 若选择“顺序不同算不同结果”，例如 10 个视频取 4 个：`P(10,4)=5040`
- 若选择“顺序不同算同一结果”，例如 10 个视频取 4 个：`C(10,4)=210`

### 6. 为什么叫“随机排列组合”，但并不是随机抽几个
- 当前实现是穷举生成（排列或组合）
- 如果后续需要，可以再迭代成“随机抽样生成”模式
