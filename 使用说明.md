# 视频混剪合并工具使用说明

这是一个用于批量拼接视频素材的桌面工具，支持两种工作方式：

- `前中后拼接`：适合固定结构的内容组合。
- `随机排列组合`：适合从一个素材池中穷举 2 / 3 / 4 段顺序组合。

---

## 功能概览

### 1. 前中后拼接
- 选择前半段目录
- 可选选择中段目录
- 选择后半段目录
- 批量生成所有组合结果

输出数量：
- 无中段时：`前半段数量 × 后半段数量`
- 有中段时：`前半段数量 × 中段数量 × 后半段数量`

### 2. 随机排列组合
- 选择一个统一素材目录
- 选择拼接数量 `2 / 3 / 4`
- 系统自动生成该目录中所有满足条件的顺序排列结果

输出数量：
- 结果总数为 `P(n, k)`
- `n` 是目录中的视频数
- `k` 是拼接数量

例子：
- 目录中有 4 个视频，拼接数量选 2：结果数为 `P(4,2)=12`
- 目录中有 4 个视频，拼接数量选 3：结果数为 `P(4,3)=24`
- 目录中有 4 个视频，拼接数量选 4：结果数为 `P(4,4)=24`

说明：顺序不同算不同结果，例如 `A+B` 和 `B+A` 是两个不同输出文件。

## 快速上手（Mac）

### 第一步：初始化环境（只需一次）
在终端进入项目目录后执行：

```bash
sh setup_mac.sh
```

脚本会准备 `bin/ffmpeg` 和 `bin/ffprobe`。

### 第二步：启动程序

```bash
./start.command
```

如果双击 `start.command`，也可以直接启动。

---

## 快速上手（Windows）

### 第一步：准备 FFmpeg
把 `ffmpeg.exe` 和 `ffprobe.exe` 放到项目根目录的 `bin/` 文件夹中。

### 第二步：启动程序
可选方式：

1. 命令行运行
```bash
python main.py
```

2. 双击运行
- 双击 `start.bat`

## 界面说明

### 顶部 Tab
- `前中后拼接`：默认模式
- `随机排列组合`：排列模式

### 素材与输出
#### 前中后拼接 Tab
- `前半段目录`：第一段素材
- `中段目录（可选）`：中间素材
- `后半段目录`：最后一段素材
- `输出目录`：结果保存目录

#### 随机排列组合 Tab
- `文件目录`：统一素材池目录
- `拼接数量`：每个结果由几个视频组成，支持 `2 / 3 / 4`
- `输出目录`：结果保存目录

### 合并选项
- `优先无损（失败自动转兼容）`
  - 先尝试无损拼接
  - 失败后自动改为兼容拼接
- `强制兼容（统一分辨率）`
  - 全部任务直接重编码

### 统一分辨率
- `自动`：以前当前任务中的第一个视频为准
- `自定义`：手动输入宽和高，例如 `1080 x 1920`

### 其他选项
- `跳过已存在文件`：输出目录中已经存在的结果不再重复生成
- `并发数`：同时运行的任务数

### 任务进度与日志
- 页面会显示总体完成进度
- 每个线程有独立日志面板
- 日志标题会显示当前文件和 ETA

## 推荐用法

### 适合用前中后拼接的场景
- 开头、中间、结尾的角色明确
- 希望固定结构批量出片
- 想控制哪些素材放在前、中、后

### 适合用随机排列组合的场景
- 素材来源统一
- 希望测试不同片段顺序的效果
- 希望快速穷举 2 / 3 / 4 段组合

---

## 性能建议

- `优先无损` 最快，但要求素材参数一致
- 素材格式较乱时，优先用 `强制兼容`
- 输出目录尽量放在 SSD
- 并发数建议从 `2-4` 起步，再按机器性能提高
- 如果随机模式目录中文件很多，结果数量会急剧增加，建议先小规模测试

## 常见问题

### 1. 提示找不到 FFmpeg
- Mac：先执行 `sh setup_mac.sh`
- Windows：确认 `bin/` 下存在 `ffmpeg.exe` 和 `ffprobe.exe`

### 2. 合并失败怎么办
- 查看页面下方线程日志
- 常见原因：文件损坏、参数不兼容、磁盘权限或输出目录问题

### 3. 无损模式黑屏或异常
- 切换到 `强制兼容（统一分辨率）`
- 兼容模式会重编码并统一输出参数

### 4. 随机模式结果数量太多
- 这是正常现象，因为当前逻辑会生成全排列
- 例如 10 个视频里取 4 个，结果数是 `P(10,4)=5040`

### 5. 为什么叫“随机排列组合”，但看起来不是随机抽几个
- 当前版本是穷举所有顺序结果，不是随机抽样
- 如果后续需要，也可以继续迭代为“随机抽样生成”模式
