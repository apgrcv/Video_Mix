# 视频混剪工具 - Mac/Windows 跨平台版

这是一个用于批量自动组合"前半段"和"后半段"视频的工具。
它支持 Mac 和 Windows，能够自动处理分辨率不一致的问题。

---

## 🚀 快速上手 (Mac 用户)

### 第一步：初始化环境（只需执行一次）
你需要下载 FFmpeg 核心组件，为了方便，我准备了一个自动脚本。

1. 打开终端 (Terminal)。
2. 进入项目目录（如果你不在的话）：
   ```bash
   cd /Users/apple/Desktop/短剧项目/新混剪工具/Video_Mix
   ```
3. 运行自动配置脚本：
   ```bash
   sh setup_mac.sh
   ```
   > 这个脚本会自动下载独立的 ffmpeg 和 ffprobe 文件到项目的 `bin/` 目录下。这样你就不需要安装任何东西，也不用担心版本问题。

### 第二步：运行与使用
在开发和日常使用中，**不需要**每次都打包。

**最简单的方法：**
1. 在文件夹中找到 `start.command` 文件。
2. 双击它。
3. 软件就会启动。

**或者在终端中运行：**
```bash
./start.command
```

> `start.command` 会自动寻找合适的 Python 环境（解决 Mac 上可能出现的 "No module named tkinter" 问题）。

## 🚀 快速上手 (Windows 用户)

### 第一步：初始化环境（只需执行一次）

1. **下载 FFmpeg**：
   - 访问 https://www.gyan.dev/ffmpeg/builds/ 下载 "release builds" 中的 "ffmpeg-release-essentials.zip"
   - 或者使用更小的 "ffmpeg-minimal-build"：
     - 从 https://github.com/BtbN/FFmpeg-Builds/releases 下载最新版本
     - 选择 `ffmpeg-master-latest-win64-gpl.zip`
   - 解压后，将 `bin/ffmpeg.exe` 和 `bin/ffprobe.exe` 复制到本项目的 `bin/` 文件夹下

2. **确保 Python 环境**：
   - Windows 通常已自带 Python，如果没有，请从 https://www.python.org/downloads/ 下载安装
   - 安装时记得勾选 "Add Python to PATH"

### 第二步：运行与使用

**方法一：直接运行**
1. 打开命令提示符 (cmd) 或 PowerShell
2. 进入项目目录：
   ```bash
   cd C:\Users\你的用户名\Desktop\短剧项目\新混剪工具\Video_Mix
   ```
3. 运行：
   ```bash
   python main.py
   ```

**方法二：使用启动脚本**
1. 双击项目中的 `start.bat` 文件
2. 软件就会启动

> **提示**：如果你在运行时遇到 "No module named tkinter" 的错误，请先安装 tkinter：
> ```bash
> pip install tkinter
> ```

## 🆕 开头目录功能

本工具支持在"前×中×后"拼接的基础上，在开头添加一个视频片段。

### 功能说明
- **开头目录**：选择一段视频作为开头
- **拼接方式**：顺序拼接（不是相乘）
- **示例**：
  - 假设 "前×中×后" 生成 90 个视频组合
  - 开头目录有 9 个视频
  - 最终结果：90 个视频，每个视频前面都按顺序添加一个开头视频
  - 第1个组合 + 开头第1个视频
  - 第2个组合 + 开头第2个视频
  - ...
  - 第10个组合 + 开头第1个视频（循环使用）

### 使用方法
1. 在软件界面中，点击"开头目录"旁的"📁 选择"按钮
2. 选择包含开头视频的文件夹
3. 确保该文件夹中有视频文件（支持 .mp4, .avi, .mov, .mkv 等格式）
4. 开始合并后，系统会自动按顺序将开头视频添加到每个组合前面

## 🛠 开发与调试指南 (回答你的疑问)

**Q: 发现 Bug 怎么调？需要每次都重新打包吗？**
**A: 完全不需要！** 
打包 (`.app` / `.exe`) 是为了发给别人用的，或者是为了你自己以后双击方便。
**在你修改代码、调试 Bug 的阶段，永远只用 `start.command` 启动。**

**调试流程：**
1. 发现问题（比如报错、功能不对）。
2. 关闭正在运行的软件窗口。
3. 修改 `main.py` 代码。
4. 双击 `start.command` 重新运行。
5. 验证问题是否解决。

只有当你觉得代码**非常完美、稳定了**，才进行“打包”操作。

## 📦 打包发布 (Mac 用户)

当你确定功能开发完成，想生成一个可以双击运行的 `.app` 文件时：

1. 安装打包工具 (只需一次)：
   ```bash
   pip3 install pyinstaller
   ```
2. 执行打包命令：
   ```bash
   pyinstaller --onefile --windowed --add-binary "bin/ffmpeg:." --add-binary "bin/ffprobe:." --name "VideoMixer" main.py
   ```
3. 等待运行完成。
4. 在 `dist/` 文件夹中，你会看到 `VideoMixer.app`。
   - 你可以将这个 `.app` 拖到应用程序文件夹，或者发给其他 Mac 用户。

## 📦 打包发布 (Windows 用户)

当你确定功能开发完成，想生成一个可以双击运行的 `.exe` 文件时：

### 方式一：使用打包脚本（推荐）

1. 双击运行项目中的 `build.bat` 脚本
2. 等待打包完成（约需1-2分钟）
3. 在 `dist/` 文件夹中会生成 `VideoMixer.exe`

**📤 发给同事时需要打包的文件：**

将 `dist/` 文件夹下的 `VideoMixer.exe` 和项目根目录下的 `bin/` 文件夹（包含 `ffmpeg.exe` 和 `ffprobe.exe`）一起发送给对方。

**正确的分享方式：**
```
📁 分享文件夹/
   ├── VideoMixer.exe    ← 主程序
   └── bin/              ← FFmpeg依赖文件夹（必须！）
       ├── ffmpeg.exe
       └── ffprobe.exe
```

对方只需要确保这两个文件在同一目录下，双击 `VideoMixer.exe` 即可运行。

### 方式二：手动打包

1. 安装打包工具 (只需一次)：
   ```bash
   pip install pyinstaller
   ```
2. 执行打包命令：
   ```cmd
   pyinstaller --onefile --windowed --add-binary "bin\ffmpeg;." --add-binary "bin\ffprobe;." --name "VideoMixer" main.py
   ```
   > 注意：Windows 使用分号 `;` 分隔路径，Mac 使用冒号 `:` 

3. 等待运行完成。
4. 在 `dist/` 文件夹中，你会看到 `VideoMixer.exe`。

**📤 发给同事时需要打包的文件：**

与方式一相同，将 `dist/VideoMixer.exe` 和 `bin/` 文件夹一起发送给同事。

## 🛠 性能优化提示

### Mac 用户
- 使用"强制兼容"模式时，系统会自动使用 **硬件加速** (VideoToolbox)
- 这可以显著提升编码速度，利用 GPU 进行视频处理

### Windows 用户
- 使用"强制兼容"模式时，系统会自动使用 **x264 优化参数**
- 预设使用 `veryfast` 模式，CRF=23 平衡画质与速度
- 确保使用最新版本的 FFmpeg 以获得最佳性能

### 通用优化建议
- **线程数**：根据 CPU 核心数设置合适的线程数（建议 4-8 个）
- **输出目录**：选择 SSD 所在的目录，可以大幅提升写入速度
- **视频格式**：如果源视频已经是 H.264 格式，使用"优先无损"模式会更快

## ⚠️ 常见问题

**1. 提示 "未找到 FFmpeg"？**
- **Mac**：请确保你运行了 `sh setup_mac.sh`，确保项目目录下有 `bin/` 文件夹
- **Windows**：请确保 `bin/` 文件夹下有 `ffmpeg.exe` 和 `ffprobe.exe`

**2. 合并失败？**
- 查看软件界面下方的"日志"窗口，它会告诉你具体的错误原因
- 常见原因是视频文件损坏，或者权限问题

**3. Windows 打包后发给同事，提示找不到 FFmpeg？**
- 这是因为只发送了 `VideoMixer.exe`，没有发送 `bin/` 文件夹
- **解决方法**：将 `VideoMixer.exe` 和 `bin/` 文件夹放在同一个文件夹下一起发送
- 或者让同事按照"快速上手 (Windows 用户)"章节手动配置 FFmpeg

**4. "优先无损"模式后半段视频看不到画面？**
- 这是已知的兼容性问题，请使用"强制兼容"模式
- 强制兼容模式会自动处理各种编码格式的兼容性问题

**5. 日志区域显示混乱？**
- 软件会根据你设置的线程数自动创建对应的日志面板
- 每个面板会显示对应线程的日志和剩余时间预估
