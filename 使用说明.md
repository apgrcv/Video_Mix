# 视频混剪合并工具使用说明

这是一个用于批量拼接视频素材的桌面工具。当前版本的主工作方式是：

- 从“前半段 / 中段（可选）/ 后半段”目录生成主体组合
- 再按顺序把“开头目录（可选）”中的视频拼接到每个结果最前面

---

## 1. 功能概览

### 前中后主体拼接
- 选择前半段目录
- 可选选择中段目录
- 选择后半段目录
- 程序自动生成所有组合结果

输出数量：
- 无中段时：`前半段数量 × 后半段数量`
- 有中段时：`前半段数量 × 中段数量 × 后半段数量`

### 开头目录顺序拼接
- 如果选择了开头目录，程序会把开头视频按顺序循环拼到每个结果前面
- 开头目录**不会增加结果数量**，只影响每个结果最前面的内容

例子：
- 前中后主体结果共 90 个
- 开头目录有 9 个视频
- 最终仍然输出 90 个视频
- 只是第 1 个结果用开头 1，第 2 个结果用开头 2，循环使用

---

## 2. 快速上手（Mac）

### 第一步：初始化环境（只需一次）
在终端进入项目目录后执行：

```bash
sh setup_mac.sh
```

脚本会准备 `bin/ffmpeg` 和 `bin/ffprobe`。

### 第二步：启动程序

```bash
./start.command
```

也可以直接双击 `start.command` 启动。

---

## 3. 快速上手（Windows）

### 第一步：准备 FFmpeg
把 `ffmpeg.exe` 和 `ffprobe.exe` 放到项目根目录的 `bin/` 文件夹中。

### 第二步：启动程序
可选方式：

1. 命令行运行
```bash
python main.py
```

2. 双击运行
- 双击 `start.bat`

---

## 4. 界面说明

### 素材与输出
当前界面包含：
- `开头目录（可选）`
- `前半段目录`
- `中段目录（可选）`
- `后半段目录`
- `输出目录`

其中：
- 前、中、后决定主体组合任务
- 开头目录决定每个结果最前面的附加片段

### 合并选项
- `优先无损（失败自动转兼容）`
  - 先尝试无损拼接
  - 失败后自动改为兼容拼接
- `强制兼容（统一分辨率）`
  - 所有任务直接重编码

### 统一分辨率
- `自动`：以前半段视频为准
- `自定义`：手动输入宽和高，例如 `1080 x 1920`

### 其他选项
- `跳过已存在文件`：输出目录中已经存在的结果不重复生成
- `并发数`：同时运行的任务数

### 任务进度与日志
- 页面会显示总体完成进度
- 每个线程有独立日志面板
- 日志标题会显示当前文件和 ETA

---

## 5. 推荐用法

### 适合当前版本的场景
- 前、中、后结构明确
- 需要大量测试不同前后段组合
- 需要在所有结果前面统一加一段或多段轮换开头

### 开头目录适合的场景
- 不想增加结果数量
- 只想在现有主体结果前面加不同开头
- 想让开头素材按顺序循环使用

---

## 6. 性能建议

- `优先无损` 最快，但要求素材参数一致
- 素材格式较乱时，优先使用 `强制兼容`
- 输出目录尽量放在 SSD
- 并发数建议从 `2-4` 起步，再按机器性能提高

## 7. 打包发布

### Mac
当功能稳定后，可执行：

```bash
pip3 install pyinstaller
pyinstaller --onefile --windowed --add-binary "bin/ffmpeg:." --add-binary "bin/ffprobe:." --name "VideoMixer" main.py
```

### Windows
可直接双击运行：
- `build.bat`

或手动执行：

```cmd
pyinstaller --onefile --windowed --add-binary "bin\ffmpeg;." --add-binary "bin\ffprobe;." --name "VideoMixer" main.py
```

打包后请把主程序和 `bin/` 一起分发。

## 8. 常见问题

### 1. 提示找不到 FFmpeg
- Mac：先执行 `sh setup_mac.sh`
- Windows：确认 `bin/` 下存在 `ffmpeg.exe` 和 `ffprobe.exe`

### 2. 合并失败怎么办
- 查看界面下方日志面板
- 常见原因：文件损坏、参数不兼容、磁盘权限、输出目录不可写

### 3. 无损模式黑屏或异常
- 切换到 `强制兼容（统一分辨率）`
- 兼容模式会重编码并统一输出参数

### 4. 为什么预计输出数量没有把开头目录算进去
- 因为开头目录不参与笛卡尔积
- 它只是顺序附加到已经生成的主体结果前面

### 5. 为什么日志区域会分成多个小面板
- 这是并发执行的线程日志展示方式
- 每个面板对应一个后台线程
