## 1. 项目概述

本项目是一个基于 Python + Tkinter 的桌面端批量视频拼接工具，核心能力是通过 FFmpeg 将多个视频按规则批量合并，生成大量成品视频。

**当前代码中的主业务模式**是：
- 可选开头目录
- 前半段目录
- 中段目录（可选）
- 后半段目录
- 输出目录

其中：
- 前、中、后使用笛卡尔积生成组合任务
- 开头目录不是相乘，而是按顺序循环拼接到每个组合前面

主要场景：短剧推广、素材批量混剪、不同前后段内容快速组合。

## 2. 当前产品功能

### 2.1 素材输入与输出

界面当前提供以下输入项：
- **开头目录（可选）**
- **前半段目录**
- **中段目录（可选）**
- **后半段目录**
- **输出目录**

统计逻辑：
- 自动统计各目录中的有效视频数量
- 自动计算预计输出数量

数量计算规则：
- 没有中段时：`前半段数量 × 后半段数量`
- 有中段时：`前半段数量 × 中段数量 × 后半段数量`
- 开头目录**不参与数量相乘**，只按顺序循环使用

### 2.2 开头目录逻辑

开头目录是当前版本仍然存在的能力：
- 若选择了开头目录，程序会在前中后拼接完成后，再把开头视频顺序拼到结果最前面
- 使用方式是循环取用：
  - 第 1 个结果使用第 1 个开头
  - 第 2 个结果使用第 2 个开头
  - ...
  - 超出后重新从第 1 个开头开始循环

说明：
- 开头目录属于顺序附加逻辑，不是新的排列组合维度
- 因此它不会影响“预计输出数量”

### 2.3 前中后拼接逻辑

当前主流程支持：
- 两段拼接：`前 + 后`
- 三段拼接：`前 + 中 + 后`

输出文件名格式：
- 两段：`front_back_merged.mp4`
- 三段：`front_middle_back_merged.mp4`
- 有开头时，最终结果会在文件名前再加开头视频名

### 2.4 合并模式

#### 1. 优先无损（失败自动转兼容）
- 先尝试无损拼接
- 条件是参与拼接的视频流参数完全一致
- 无损失败时自动回退到兼容模式
- 无损模式使用 `-fflags +genpts`，用于修复时间戳兼容问题

#### 2. 强制兼容（统一分辨率）
- 无论源视频参数是否一致，都直接重编码
- 适用于分辨率、编码、帧率不一致的素材

### 2.5 分辨率策略

#### 自动
- 以前半段视频分辨率为基准

#### 自定义
- 用户手动输入目标宽和高，例如 `1080 x 1920`

#### 画面处理方式
- 使用 `scale + pad + setsar=1 + format=yuv420p`
- 保持比例，不拉伸
- 黑边补齐画布
- 提高不同素材之间的兼容性

### 2.6 音频策略
- 所有输入片段都有音频时：拼接音频和视频
- 只要任意片段缺少音频：输出纯视频结果

### 2.7 并发与日志
- 使用 `ThreadPoolExecutor` 做并发执行
- 并发数由用户设置，默认取保守值（2 到 4）
- 日志区域会按并发数拆成多个线程面板
- 每个面板显示：
  - 当前处理文件
  - 当前线程日志
  - ETA 倒计时
- 点击“停止任务”时，会终止正在运行的 FFmpeg 进程，并尝试删除未完成输出文件

### 2.8 跨平台能力
- **macOS**：优先使用 `h264_videotoolbox` 做硬件编码
- **Windows / Linux**：使用 `libx264`，参数为 `veryfast + crf 23`
- 打开输出目录时自动适配 Finder / Explorer / xdg-open

## 3. 技术实现要点

### 3.1 技术栈
- 语言：Python 3.10+
- GUI：Tkinter
- 视频处理：FFmpeg / FFprobe
- 并发：`ThreadPoolExecutor`
- 路径处理：`pathlib.Path`

### 3.2 当前核心文件
- `main.py`
  - GUI 主程序
  - 素材统计
  - 前中后任务生成
  - 开头目录顺序拼接
  - 并发调度
  - FFmpeg 调用
- `cli.py`
  - 早期命令行版本
  - 当前能力少于 GUI，不代表完整产品形态
- `使用说明.md`
  - 面向使用者的说明文档
- `技术方案.md`
  - 面向后续开发的简版技术说明

### 3.3 拼接内核
当前主程序已经统一支持：
- 2 段拼接
- 3 段拼接
- 带开头的二次顺序拼接

核心方法包括：
- `concat_copy()`
- `concat_copy_three()`
- `concat_reencode()`
- `concat_reencode_three()`

### 3.4 任务生成流程
1. 扫描开头、前半段、中段、后半段目录
2. 根据前中后目录生成笛卡尔积任务
3. 若勾选“跳过已存在文件”，则先过滤已存在结果
4. 先完成主体拼接
5. 若选择了开头目录，再执行开头顺序拼接
6. 把任务提交给线程池执行，并更新完成数、日志和 ETA

## 5. 当前实现边界

- 当前没有“随机排列组合”模式
- 当前没有顶部 Tab 切换
- 当前 UI 仍基于原生 Tk 外观，没有引入额外主题库
- 输出文件名使用素材文件名 `stem` 拼接，如果不同文件仅扩展名不同且主名相同，存在文件名冲突风险
- `cli.py` 仍然是旧逻辑，不代表 GUI 的完整能力
