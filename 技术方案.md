视频混剪合并工具技术方案

一、目标
- 支持批量视频拼接
- 支持两种任务模式：前中后拼接、随机排列组合
- 支持 2 / 3 / 4 段视频合并
- 支持可选轻量转场
- 跨平台运行，Mac 与 Windows 均可用
- 处理分辨率不一致，输出统一尺寸

二、技术选型
- 语言：Python 3
- UI：Tkinter + ttk.Notebook
- 合并引擎：FFmpeg + FFprobe
- 并发：ThreadPoolExecutor
- 打包：PyInstaller

三、核心流程
1) 根据当前 Tab 选择任务模式
2) 扫描素材目录并筛选视频文件
3) 计算任务数量
   - 前中后拼接：笛卡尔积
 - 随机排列组合：
   - 顺序不同算不同结果：排列数 `P(n, k)`
   - 顺序不同算同一结果：组合数 `C(n, k)`
4) 对每个任务执行拼接
   - 未启用转场：优先无损，失败则重编码
   - 启用转场：先做片段级预处理，再走快速 concat 拼接
5) 输出到指定目录并记录日志、进度、ETA

四、拼接策略
- 支持 2 / 3 / 4 段统一处理
- 无损拼接要求流参数一致
- 普通重编码拼接使用动态 filter_complex 生成 concat 链
- 轻量转场使用“素材标准化一次 + 片段边界预处理 + 快速 concat”方案
- 输出文件名按素材顺序拼接生成

五、转场策略
- UI 侧支持：
  - 开关
  - 轻量转场时长选择
- 轻量转场：
  - 对相邻片段边界做 `fade/afade`
  - 在两段之间插入极短黑场过渡片段
  - 总时长支持 0.4 / 0.6 / 0.8 / 1.0 秒
- 允许缓存标准化结果和边界片段，避免重复编码同一素材

六、统一分辨率与兼容策略
- 自动模式：以前当前任务中的第一个视频为目标尺寸
- 自定义模式：用户手动输入目标宽高
- 普通兼容处理：
  - scale + pad + setsar=1 + format=yuv420p
- 转场处理：
  - scale + pad + fps=30 + settb=AVTB + setsar=1 + format=yuv420p

七、音频策略
- 所有输入都有音频：拼接音频与视频
- 任一输入无音频：输出无音频视频
- 启用转场时，片段音频边界会做 `afade` 处理；轻量模式的黑场片段附带短静音音轨

八、UI 结构
- Tab1：前中后拼接
  - 前半段目录
  - 中段目录（可选）
  - 后半段目录
- Tab2：随机排列组合
  - 文件目录
  - 拼接数量（2 / 3 / 4）
  - 拼接顺序（区分顺序 / 不区分顺序）
- 共享区域：
  - 输出目录
  - 合并模式
  - 统一分辨率
  - 跳过已存在文件
  - 转场效果
  - 并发数
  - 任务进度
  - 运行日志
- 交互细节：
  - 鼠标滚轮滚动时，Notebook Tab 不随滚轮切换，避免误触

九、已知边界
- 随机排列组合不是随机抽样，而是穷举生成（排列或组合取决于拼接顺序设置）
- 启用转场后仍会慢于无损，但已从“整段重叠滤镜”切换为“边界预处理快拼接”路线
- 片段时长过短时，转场时长会被压缩，极端情况下会直接失败
- macOS 下启用转场时，当前实现为稳定性优先，改用 `libx264`
- 文件名基于 stem 生成，存在同名素材时可能冲突
- CLI 版本尚未同步新模式与转场能力

十、可扩展方向
- 随机抽样生成
- 结果数量上限控制
- 每个拼接点随机不同转场
- 更多转场特效
- 失败任务重试
- 断点续跑
- 输出命名规则配置
- CLI 双模式与转场同步
