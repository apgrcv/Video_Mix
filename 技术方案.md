视频混剪合并工具技术方案

一、目标
- 支持批量视频拼接
- 支持两种任务模式：前中后拼接、随机排列组合
- 支持 2 / 3 / 4 段视频合并
- 跨平台运行，Mac 与 Windows 均可用
- 处理分辨率不一致，输出统一尺寸

二、技术选型
- 语言：Python 3
- UI：Tkinter + ttk.Notebook
- 合并引擎：FFmpeg + FFprobe
- 并发：ThreadPoolExecutor
- 打包：PyInstaller

三、核心流程
1) 根据当前 Tab 选择任务模式
2) 扫描素材目录并筛选视频文件
3) 计算任务数量
   - 前中后拼接：笛卡尔积
   - 随机排列组合：排列数 `P(n, k)`
4) 对每个任务执行拼接
   - 优先无损：concat demuxer + copy
   - 无损失败或强制兼容：重编码 + 统一分辨率
5) 输出到指定目录并记录日志、进度、ETA

四、拼接策略
- 支持 2 / 3 / 4 段统一处理
- 无损拼接要求流参数一致
- 重编码拼接使用动态 filter_complex 生成 concat 链
- 输出文件名按素材顺序拼接生成

五、统一分辨率策略
- 自动模式：以前当前任务中的第一个视频为目标尺寸
- 自定义模式：用户手动输入目标宽高
- 兼容模式处理方式：
  - 使用 scale + pad 保持比例
  - setsar=1
  - format=yuv420p
  - 输出统一画布大小

六、音频策略
- 所有输入都有音频：拼接音频与视频
- 任一输入无音频：输出无音频视频

七、UI 结构
- Tab1：前中后拼接
  - 前半段目录
  - 中段目录（可选）
  - 后半段目录
- Tab2：随机排列组合
  - 文件目录
  - 拼接数量（2 / 3 / 4）
- 共享区域：
  - 输出目录
  - 合并模式
  - 统一分辨率
  - 并发数
  - 任务进度
  - 运行日志

八、已知边界
- 随机排列组合当前是“全排列穷举”，不是随机抽样
- 文件名基于 stem 生成，存在同名素材时可能冲突
- CLI 版本尚未同步新模式

九、可扩展方向
- 随机抽样生成
- 结果数量上限控制
- 失败任务重试
- 断点续跑
- 输出命名规则配置
