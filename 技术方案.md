视频混剪合并工具技术方案

一、目标
- 支持批量视频拼接
- 支持开头 + 前中后的组合流程
- 支持两段与三段主体拼接
- 跨平台运行，Mac 与 Windows 均可用
- 处理分辨率不一致，输出统一尺寸

二、技术选型
- 语言：Python 3
- UI：Tkinter（原生库）
- 合并引擎：FFmpeg + FFprobe
- 并发：ThreadPoolExecutor
- 打包：PyInstaller

三、核心流程
1) 扫描开头、前半段、中段、后半段目录并筛选视频文件
2) 计算主体任务数量
   - 无中段：前 × 后
   - 有中段：前 × 中 × 后
3) 对每个主体任务执行拼接
   - 优先无损：concat demuxer + copy
   - 无损失败或强制兼容：重编码 + 统一分辨率
4) 若存在开头目录，则按顺序把开头视频拼接到主体结果前面
5) 输出到指定目录并记录日志、进度、ETA

四、拼接策略
- 支持两段与三段主体拼接
- 支持在主体结果前追加一个开头视频
- 无损拼接要求流参数一致
- 重编码拼接使用 filter_complex concat 链
- 输出文件名按素材顺序拼接生成

五、统一分辨率策略
- 自动模式：以前半段视频为目标尺寸
- 自定义模式：用户手动输入目标宽高
- 兼容模式处理方式：
  - 使用 scale + pad 保持比例
  - setsar=1
  - format=yuv420p
  - 输出统一画布大小

六、音频策略
- 所有输入都有音频：拼接音频与视频
- 任一输入无音频：输出无音频视频

七、UI 结构
- 素材区：
  - 开头目录（可选）
  - 前半段目录
  - 中段目录（可选）
  - 后半段目录
  - 输出目录
- 选项区：
  - 合并模式
  - 统一分辨率
  - 跳过已存在文件
  - 并发数
- 监控区：
  - 任务进度
  - ETA
  - 运行日志

八、已知边界
- 当前没有顶部 Tab 切换
- 当前没有随机排列组合模式
- 文件名基于 stem 生成，存在同名素材时可能冲突
- CLI 版本尚未同步 GUI 的完整流程
- 当前 UI 仍是原生 Tk 外观

九、近期讨论但未落地的方向
- 前中后 / 随机排列组合双 Tab
- 基于 `P(n, k)` 的排列任务生成
- 更明显的 UI 视觉升级

十、可扩展方向
- 重新实现随机排列组合模式
- 结果数量上限控制
- 失败任务重试
- 断点续跑
- 输出命名规则配置
- CLI 升级
